League Champion Explorer

Strategies: Development

Timeline: Nov-Dec 2020

Tools Used: React, Node.js

The Problem

React is one of the most popular JavaScript front-end libraries today. I believe the ability to use tools such as React and Node.js is just as important as the ability to create intuitive interfaces.

Therefore, for this project, I both designed and implemented my application. My goal was to create a list interface for a website that supported several features. Specifically, I wanted to support the following:

  1. Filtering by Categories
  2. Sorting by Categories
  3. Aggregating Selected Items
  4. Displaying Items and Associated Data

Creating The Application

The first goal of my website was to create an application where users could explore League of Legends characters and decide which ones to buy. League of Legends is a popular online game that has (as of December 2020) 152 unique characters. With so many choices, it can be difficult for new players to decide what characters they might want to buy.

In an attempt to solve this problem, my application provides convenient filtering and sorting tools so a new user can easily find a champion they are looking for, and provides a cart feature so that the user can see the total cost of all the characters they wish to purchase. Filters and sorting can be combined in any number of ways so users have the freedom to filter to the granularity that they require.

My other main goal was to create a React application with a clean and intuitive architecture that future developers would also be able to build off of. Therefore, while writing the code for my web app, I tried to follow best practices, such as separating distinct functionality into separate React components and commenting my code where necessary.

Design Principles

Although the primary goal of this project was more development focused and less design focused, I still followed best design principles while creating my application.

Visual Consistency
  1. In character cards, the button is left left-aligned with the text.
  2. Class and Role Icons use the same, consistent gold color scheme.
  3. All buttons are aligned to the bottom of each character card. This also makes it easy to quickly add multiple characters to the cart.
Color
  1. I chose a dark themed color palette with sufficient contrast.
  2. I also chose a bright blue as my primary color to highlight certain interface elements like buttons.
Layout
  1. The most important content, the character cards, are in the center of the page.
  2. The filter/sort menu and the cart are large and placed at the sides of the page. Their large size makes them easy to navigate to from anywhere on the page (Fitts's Law).
  3. The page is responsive and the grid adapts to the size of the viewport.
Typography
  1. I chose to use a native font stack for my application.
  2. I use different text sizes to highlight importance. Larger text is more important in my app.

Takeaways

Developing a website using React and JavaScript instead of basic HTML/CSS is definitely a much different experience. In general, I felt that using React was a more "traditional" programming experience. Using React components allowed me to create reusable pieces of code similar to object-oriented programming. If I was using only HTML and CSS, I would have been forced to simply copy and paste my code.

Although developing a React application definitely has a learning curve, as you need to learn React specific syntax and rules, I think the overall development experience with React is much more enjoyable that using the typical HTML and CSS combination. HTML/CSS/JavaScript are good for basic websites, but implementing more advanced features that could be reused such as a list interface with filtering and sorting would be much more difficult than doing the same using React.
