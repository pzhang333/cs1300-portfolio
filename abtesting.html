A/B Testing Experiment

Strategies: User Research

Timeline: Oct 2020

Tools Used: Python, Statistics

The Problem

Visual design is a polarizing subject and is often subject to personal biases. However, there are times when we need to be able to defintely say "Design A is better than Design B." A/B testing and statistical analysis are powerful tools that can help us to determine which designs are better.

For this experiment, I first created 2 versions of the same shopping site. Users of the website were given a single task: add 150 dollars worth of cacti to the shopping cart.

Forming Hypotheses

For this experiment, I was interested in 2 specific metrics. The first metric was time to completion and the other metric was return rate.

After specifying which metrics I was interested in, I then created a hypotheses for each metric. In statistics, a hypothesis is actually composed of 2 things: a null hypothesis and an alternative hypothesis. A null hypothesis takes the position that there is no difference between versions A and B of my website, while an alternative hypothesis claims there is a difference. I list my hypotheses below.

Time to Completion:

Null: There is no significant difference in time to completion between Site A and Site B.

Alternative: Time to completion will be shorter for Site B because the grid layout makes finding different items easier and therefore should reduce the time needed to complete the task.

Return Rate:

Null: There is no significant difference in return rate between Site A and Site B.

Alternative: There is a difference in return rate between Site A and Site B because Site A highlights quantities and prices in red, which should reduce how often a user needs to check the cart to get the total price.

All About Data

The process of data collection can be split into 3 distinct parts. First, the actual collection of data. Next, the cleaning of data. And finally, the processing of cleaned data.

Data Collection

To collect my data, I added listeners to each sites' buttons. When a button was clicked by a user, a new log line would be generated and then stored for later processing.

Data Cleaning

Once I have my raw data logs, I need to clean my data. This is an important step because bad data can skew my final results. An example of a bad log is one that shows a user who loaded the site but did not interact with my site. Logs like this can result in false conclusions unless they are removed from the data set.

Data Analysis

The final step is to analyze my clean data, which I did using custom Python scripts that I previously wrote. I can then use the outputs from my Python scripts to reach conclusions about my experiment.

Takeaways

  1. Having a sufficient quantity of data is extremely important. With the relatively limited number of users in my data set, it is hard to get a statistically significant result. I think that if I had more users and thus more data, I might have been able to obtain different results.
  2. It will be easier to get statistically significant results using big design changes as opposed to smaller design changes. As a whole, the differences between my Site A and Site B definitely existed, but they were not drastic, which might explain why I did not obtain statistically significant results for either metric.
  3. Having a diverse pool of new users is important. Examining the data points individually, I found that most users had similar completion times. I hypothesize that part of this can be attributed to familiarity with the task. I used other students in my class to test my websites, and other students did the same. The result of this is that every student new exactly which buttons to click to reach 150 dollars worth of cacti. This skew the results since the users are not representative of real users.

Although I am disappointed that I did not obtain statistically significant results for either of my metrics, I still found that this assignment was a great learning opportunity. I think A/B testing and statistical analysis are very important tools, and I will definitely use them in the future to analyze my designs.
